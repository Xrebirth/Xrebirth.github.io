第1节：数据库基本使用

数据库结构

DBDBMSSQL
DataBase（数据库，数据库实际上在硬盘上以文件的形式存在）DataBase Management System（数据库管理系统，常见的有：MySQL Oracle DB2 Sybase SqlServer…）结构化查询语言，是一门标准通用的语言。标准的sql适合于所有的数据库产品。
SQL属于高级语言。只要能看懂英语单词的，写出来的sql语句，可以读懂什么意思。
SQL语句在执行的时候，实际上内部也会先进行编译，然后再执行sql。（sql语句的编译由DBMS完成。）
  • DB：数据库
  • DBMS：数据库管理系统：DBMS负责执行sql语句，通过执行sql语句来操作DB当中的数据。
  • SQL：数据库执行语句

执行过程：SQL(语句) -(输入)-> DBMS-(执行SQL语句)-> DB(通过SQL语句管理数据库)

SQL分类

DQL（数据查询语言）DML（数据操作语言）DDL（数据定义语言）TCL（事务控制语言）DCL（数据控制语言）
查询语句，凡是select语句都是DQL。insert、delete、update，对表当中的数据进行增删改。create、drop、alter，对表结构的增删改。commit提交事务，rollback回滚事务。(TCL中的T是Transaction)grant授权、revoke撤销权限等。

增删改查有一个术语：CRUD操作
Create（增） Retrieve（检索） Update（修改） Delete（删除）

表结构

  1. 表：table是数据库的基本组成单元，所有的数据都以表格的形式组织，目的是可读性强。
  2. 一个表包括行和列：
    • 行：被称为数据/记录(data)
    • 列：被称为字段(column)
  3. 每一个字段应该包括哪些属性？
    • 字段名
    • 数据类型
    • 相关的约束

表查询

条件查询

语法：

select 字段名1 ‘别名’, 字段名2 ‘别名’ from 表名 where 查询条件;

模糊查询

语法：

select 字段名1 ‘别名’, 字段名2 ‘别名’ from 表名 where 字段名 like 查询条件;

_t%指：查询第二个为t的数据

_/_指：查询第二个为/的数据(其中”/“为转义符)

%指：任意位置

数据排序

语法：

select 字段名1 ‘别名’, 字段名2 ‘别名’ from 表名 order by 字段名; // 不写默认为升序
select 字段名1 ‘别名’, 字段名2 ‘别名’ from 表名 order by 字段名 asc; // 升序
select 字段名1 ‘别名’, 字段名2 ‘别名’ from 表名 order by 字段名 desc; // 降序

select 字段名1 ‘别名’, 字段名2 ‘别名’ from 表名 order by 字段1 desc,字段2 asc,...; // 降序

注：添加两个及两个以上排序字段的话mysql会按越靠前的字段越能起到主导作用。只有当前面的字段无法完成排序的时候**(前面两个字段都相同时)**，才会启用后面的字段。

分组函数

多行函数

多行函数：在一个表中数据多行数据集中计算称之为多行函数。

概括：输入多行，输出一行。

常用函数：
  1. count() 计数
  2. sum() 求和
  3. avg() 平均值
  4. max() 最大值
  5. min() 最小值

注：1. 以上函数会自动忽略null值。
2.sql语法规定：以上函数不能不出现在where关键子后面。

count(*)和count(字段名)的区别：

count(*)：不是统计某个字段中的个数，而是统计总记录数。

count(字段名)：表示统计此字段非null的总数量。

单行函数

单行函数：在一个表中的某行计算元素。

概括：输入一行，输出一行。

注：在sql计算中数字与null运算均为null

分组查询关键字：
  1. group by： 按照某个字段或者某些字段进行分组。

SQL执行关键字顺序：

  • select –>5

    …(显示哪几条数据)

  • from –>1

    …(查询哪个表)

  • where –>2

    …(查询条件)

  • like –>2

    …(模糊查询)

  • group by –>3

    …(分组函数查询)

  • having –>4

    …(过滤分组查询数据)

  • order by –>6

    …(排序输出)

  • limit –>7
    …(取部分数据)

注：函数不能在where中使用，因为函数是在where之后执行的。
当一条语句中有 group by的话, select后面只能跟分组函数和参与分组的字段。

  1. having ：是对函数分组之后的数据进行二次过滤。

    注：当where无法实现的时候在使用having。

常用函数：
  • ifnull(可能为null的数据,被当做什么处理) 判断字段是否为null

其他关键字

distinct：关键字去除重复记录

注：以上关键子要放在所有字段名的前面
select distinct 字段名 from 表名;

外连接：

内外连接区别：

  • 内连接：

    假设A和B表进行连接，使用内连接的话，凡是A表和B表能够匹配上的记录查询出来，这就是内连接。
    特点：AB两张表没有主副之分，两张表是平等的。

  • 外连接：

    假设A和B表进行连接，使用外连接的话，AB两张表中有一张表是主表，一张表是副表，主要查询主表中的数据，捎带着查询副表，当副表中的数据没有和主表中的数据匹配上，副表自动模拟出NULL与之匹配。

  • 外连接的分类

    左外连接（左连接）：表示左边的这张表是主表。
    右外连接（右连接）：表示右边的这张表是主表。

    注：左连接有右连接的写法，右连接也会有对应的左连接的写法。

char与varchar的区别：

在实际开发中，当某个字串中的数据长度不发生改变的时候，是定长的，例：性别、生日都是采用char，当一个字段数据长度不确定，例如：简介、姓名都是采用carchar。

注意：像BLOB与CLOB这种数据类型是不可以在吗MySQL中直接输入的，需要使用Java中的IO流。

约束

概述：

在创建表的时候，可以给表的字段添加相应的约束，添加约束的目的是为了保证表中数据的合法性、有效性、完整性。

列级约束

列级约束：当前字段添加的某个约束

表级约束

表级约束：多个字段联合起来添加某个约束

主键约束

主键相关术语：
  • 主键约束：primary key
  • 主键字段：id字段添加primary key之后，id叫做主键字段
  • id字段中的每一个值都是主键值
主键作用：
  • 表的设计三范式中有要求，第一范式就要求任何一张表都应该有主键。
  • 主键的作用：主键值是这行记录在这张表当中的唯一标识。(就像一个人的身份证号一样)。
主键的分类：
根据主键字段数量划分：
  • 单一主键(常用，推荐) 又名：列级约束
  • 复合主键(多个字段联合起来添加一个主键约束) 又名：表级约束
    注：复合主键不建议使用，因为复合主键违背三范式。
根据主键性质划分：
  • 自然主键：在主键字段中不和系统的业务挂钩，称为自然主键。
  • 业务主键：主键值和系统的业务挂钩，例如：拿着银行卡的卡号做主键,拿着身份证号码作为主键。(不推荐用)注：最好不要拿着和业务挂钩的字段作为主键，因为以后的业务一旦发生改变的时候，主键值可能也需要随着发生改变，但有的时候没办法发生变化。
  • 注：一张表的主键约束只能有一个。

外键约束：

外键相关术语：
  • 外键约束：foreign key
  • 外键字段：添加有外键约束的字段
  • 外键值：外键字段中的每一个值。
注意(顺序要求)：

  • 创建表时，先创建父表，在创建子表。

  • 删除表时，先删除子表，在删除父表。

  • 删除数据时，先删除子表，在删除父表。

  • 添加数据时，先添加父表，在添加子表。

  • 注：子表绑定父表中的字段最低要求必须是：父表中的字段必须有唯一标识(绑定的父表中的字段可以不是主键)

业务背景：
请设计数据库表,用来维护学生和班级的信息?
第一种方案: 一张表存储所有数据
no(pk)    name    classno        classname
----------------------------------------------
1       zs1      101        衡水一中高中一班
2       zs2      102        衡水一中高中一班
3       zs3      103        衡水一中高中一班
4       zs4      104        衡水一中高中二班
5       zs5      105        衡水一中高中二班
以上表存在缺点：冗余。

第二种方案: 两张表(班级表和学生表)
t_class 班级表    (父表)
cno(pk)        cname    
----------------------------------------------
101     衡水一中高中一班
102     衡水一中高中二班
t_student 学生表 (子表)
sno(pk)  sname           cno(该字段添加外键约束fk)
----------------------------------------------
1         zs1          101        
2         zs2          101        
3         zs3          101        
4         zs4          102        
5         zs5          102        
以上两张表t_class为父表，t_student为子表。
建表语句:
        drop table if exists t_student;
        drop table if exists t_class;

        create table t_class(
            cno int,
            cname varchar(255),
            primary key(cno)
        );

        create table t_student(
            sno int,
            sname varchar(255),
            classno int,
            foreign key(classno) references t_class(cno)    // 绑定父表中的外键约束语句
        );

        insert into t_class values(101,'衡水一中高中一班');
        insert into t_class values(102,'衡水一中高中二班');

        insert into t_student values(1,'zs1',101);
        insert into t_student values(2,'zs2',101);
        insert into t_student values(3,'zs3',101);
        insert into t_student values(4,'zs4',102);
        insert into t_student values(5,'zs5',102);
        select * from t_class;
        select * from t_student;
常见约束解释
not null非空约束：约束的字段不能为NULL
unique唯一约束：约束的字段不能重复
primary key主键约束：约束的字段既不能为NULL，也不能重复（简称PK）
foreign key外键约束：…（简称FK）
check检查约束：注意Oracle数据库有check约束，但是mysql没有，目前mysql不支持该约束。

存储引擎(了解)

引例

        // 完整的建表语句
        CREATE TABLE `t_x` (
          `id` int(11) DEFAULT NULL
        ) ENGINE=InnoDB DEFAULT CHARSET=utf8;

观察上表中的语句可得：

  • mysql默认使用的存储引擎是InnoDB方式。

  • 默认采用的字符集是UTF8。

概述

存储引擎这个名字只有在MySQL中存在。（Oracle中有对应的机制，但是不叫做存储引擎。Oracle中没有特殊的名字，就是“表的存储方式”），mysql支持很多存储引擎，每一个存储引擎都对应了一种不同的存储方式。每一个存储引擎都有自己的优缺点，需要在合适的时机选择合适的存储引擎。

查看当前MySQL中支持的存储引擎语句：show engines \G

常见的存储引擎：

            Engine: MyISAM
                  Support: YES
                  Comment: MyISAM storage engine
            Transactions: NO
                         XA: NO
              Savepoints: NO
            
            MyISAM这种存储引擎不支持事务。
            MyISAM是mysql最常用的存储引擎，但是这种引擎不是默认的。
            MyISAM采用三个文件组织一张表：
                xxx.frm（存储格式的文件）
                xxx.MYD（存储表中数据的文件）
                xxx.MYI（存储表中索引的文件）
            优点：可被压缩，节省存储空间。并且可以转换为只读表，提高检索效率。
            缺点：不支持事务。

        -----------------------------------------------------------------------------

                  Engine: InnoDB
                  Support: DEFAULT
                  Comment: Supports transactions, row-level locking, and foreign keys
            Transactions: YES
                         XA: YES
              Savepoints: YES
                
            优点：支持事务、行级锁、外键等。这种存储引擎数据的安全得到保障。
            
            表的结构存储在xxx.frm文件中
            数据存储在tablespace这样的表空间中（逻辑概念），无法被压缩，无法转换成只读。
            这种InnoDB存储引擎在MySQL数据库崩溃之后提供自动恢复机制。
            InnoDB支持级联删除和级联更新。
        
        -----------------------------------------------------------------------------

                    Engine: MEMORY
                  Support: YES
                  Comment: Hash based, stored in memory, useful for temporary tables
            Transactions: NO
                         XA: NO
              Savepoints: NO
            
            缺点：不支持事务。数据容易丢失。因为所有数据和索引都是存储在内存当中的。
            优点：查询速度最快。
            以前叫做HEPA引擎。
